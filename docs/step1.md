近未来SF風の架空企業「Novamnesis」のランディングページを作成してください。

正式なサービス名は「Novamnesis — Memory Experience Marketplace」とします。

Novamnesis は、人間の体験記憶を記録・匿名化・編集・安全処理し、他のユーザーが体験できる「記憶商品」として販売する、架空の記憶体験マーケットプレイスです。

訪問者は、人工記憶や編集済みの体験記憶によって「冒険」「成功」「恋愛」「宇宙旅行」「スパイ任務」「極限体験」など、現実では体験できない願望を記憶として体験できます。

また、ユーザー自身の実体験記憶を Novamnesis に登録・査定・売却することもできます。

ただし、既存映画や実在作品の固有名詞・ロゴ・キャラクター・デザインは使わないでください。
あくまでオリジナルの架空企業サイトとして作ってください。
実在する医療・金融・法務サービスのように誤認されないよう、フッターに「本サイトは架空のデモサイトです」と明記してください。

目的：
訪問者に「人生を変えるような記憶体験を購入できる」と感じさせると同時に、「自分の記憶にも市場価値がある」と思わせる、近未来的で少し怪しいが魅力的なランディングページを作る。

技術要件：
- React + TypeScript で作成
- Tailwind CSS を使用
- 1ページ完結のランディングページ
- レスポンシブ対応
- ダークテーマ基調
- 未来的なUI
- アニメーションは控えめに入れる
- 実際の決済や個人情報送信は不要
- CTAボタンを複数配置する
- CTAボタンのクリック時は alert 表示程度でよい
- 外部APIは使わない
- 画像素材は使わず、CSSグラデーションや図形で未来的な雰囲気を作る

デザイン方向性：
- 近未来SF
- 高級感のある企業サイト
- 少しディストピア感
- 静かな不穏さ
- ネオン、ガラスモーフィズム、ホログラム風
- 背景は暗め
- アクセントカラーはシアン、紫、マゼンタ系
- 安っぽいゲーム画面ではなく、Apple製品サイトのような洗練感も少し入れる
- 「便利そうだが倫理的に怖い」と感じる雰囲気にする
- あからさまなホラーにはしない

ページ全体のコピー方針：
- 日本語で作成
- 少し怪しいが魅力的
- 直接的に怖すぎない
- 企業サイトとして成立する文体
- SF映画っぽさはあるが、既存作品名は出さない
- 「記憶」「現実」「もうひとつの人生」「選べる過去」「体験済みの未来」「記憶資産」「匿名化された人生」などの言葉をうまく使う

UI要件：
- 上部に固定ヘッダー
- ロゴ風テキスト「Novamnesis」
- サブタイトルとして「Memory Experience Marketplace」
- ナビゲーション：
  - Experience
  - Marketplace
  - Sell Memory
  - Safety
  - FAQ
- ヒーローには抽象的な脳波・記憶データ・ホログラム風のビジュアルをCSSで表現
- プランカードはホバー時に少し浮き上がる
- CTAボタンはグラデーション
- セクション間に十分な余白を取る
- モバイルでも読みやすくする

ページ構成：

1. ヒーローセクション

キャッチコピー：
「記憶は、もう資産になる。」

サブコピー：
「Novamnesis は、人間の体験記憶を記録・匿名化・再構成し、誰かの“もうひとつの人生”として届ける記憶体験マーケットプレイスです。」

CTA：
- 「記憶を体験する」
- 「記憶を売却する」

補足コピー：
「退屈な人生に、もうひとつの記憶を。」
「あなたが体験しなかった過去を、記憶として手に入れる。」

2. 課題提起セクション

以下のような内容を入れてください。

- 毎日同じ通勤、同じ仕事、同じ夜。
- 本当に欲しかった人生は、まだ体験できていない。
- 一方で、あなたがすでに体験した人生の一部は、誰かにとって強く望まれる記憶かもしれない。
- 記憶は、消えていくものではなく、保存され、編集され、流通する資産になりつつある。

見出し例：
「あなたの人生は、本当にこれだけですか？」

3. サービス説明セクション

Novamnesis のサービスを説明してください。

内容：
- 人工記憶や編集済みの実体験記憶によって、現実ではなく「記憶として」特別な人生を体験できる
- 体験は数十分でも、記憶上は数日・数週間の冒険として残る
- 記憶は感情強度、物語性、感覚情報、倫理審査レベルに基づいて調整される
- 体験前にカウンセリングと記憶整合性チェックを行う
- 体験後には現実復帰のための認知安定プロトコルを行う

4. Memory Marketplace セクション

Novamnesis が記憶のマーケットプレイスであることを説明してください。

内容：
- 人の体験記憶を匿名化・編集・再構成して販売する
- 購入者は、他人の人生の一部や、編集済みの冒険・成功・恋愛・旅・極限体験を記憶として体験できる
- 提供者は、自分の記憶を資産として登録できる
- 記憶はそのまま販売されるのではなく、個人情報や第三者情報を除去したうえで、体験商品として再構成される
- すべての記憶には、没入度・感情強度・現実混同リスクなどのレーティングが付く

見出し例：
「人生は、共有できる資産になる。」

実装方針：
- コンポーネント分割して作成してください
- App.tsx で完結してもよいが、可読性を優先してください
- ダミーデータは配列で管理してください
- 体験プラン、査定基準、買い取りカテゴリ、FAQ、レビューは配列データから描画してください
- 外部APIは使わないでください
- 画像素材は使わず、CSSグラデーションや図形で未来的な雰囲気を作ってください
- 可能なら軽いフェードインアニメーションを入れてください
- framer-motion が使える環境なら使ってもよいですが、必須ではありません
- lucide-react が使える環境ならアイコンに使ってもよいですが、必須ではありません
- 依存関係を増やしすぎないでください
