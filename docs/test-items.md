# Webサイト テスト項目一覧

この一覧では、テストを次の2種類に分けます。

- ページ別テスト: 各URL単体で、表示内容・操作・状態変化が正しいか確認する
- 導線テスト: 複数ページをまたいで、ユーザーが目的の画面まで迷わず進めるか確認する

補足: 厳密にはReactコンポーネント単位のテストを「単体テスト」、ブラウザでページや導線を確認するものを「E2Eテスト」または「結合テスト」と呼ぶことが多いです。ただし実務上の管理表としては「テスト項目一覧」で問題ありません。

## 1. ページ別テスト

## 2. 導線テスト

| ID | シナリオ | 手順 | 期待結果 |
|---|---|---|---|
| F-001 | トップから予約完了まで進む | `/` → カタログ導線 → `/catalog` → テンプレート詳細 → 予約CTAまたはヘッダー予約 → `/booking` → 必須項目入力 → 送信 | 予約完了画面が表示され、入力内容が反映される |
| F-002 | 体験開始から予約へ進む | `/experience` → 体験タイプを選択 → 予約へ進む → `/booking?memoryType=...` | 予約ページで選択した体験タイプが初期値になる |
| F-003 | カタログからテンプレート詳細を確認して戻る | `/catalog` → 任意のテンプレート詳細 → カタログへ戻る | `/catalog#templates` に戻れる |
| F-004 | カタログからエピソード詳細を確認して戻る | `/catalog` → 任意のエピソード詳細 → カタログへ戻る | `/catalog#episodes` に戻れる |
| F-005 | 施設確認からFAQへ進む | `/facility` → セッション動画を開閉 → アフターケアのFAQリンク | 動画モーダルが正常に開閉し、`/faq` に遷移する |
| F-006 | 売却ページ内のアンカー導線を確認する | `/sell` → 査定基準を見る → 売却プロセスを見る | それぞれ該当セクションへ移動する |
| F-007 | キャンペーンから通常査定へ進む | `/campaign/memory-buyback` → 通常査定ページへ進む | `/sell#valuation` に遷移する |
| F-008 | キャンペーンから相談へ進む | `/campaign/memory-buyback` → 相談してから申し込む → 問い合わせフォーム送信 | `/contact` に遷移し、問い合わせ受付完了画面が表示される |
| F-009 | 安全性確認後に予約する | `/safety` → ヘッダーのBooking → `/booking` → 必須項目入力 → 送信 | 安全性ページから予約完了まで進める |
| F-010 | FAQ確認後に予約する | `/faq` → ヘッダーのBooking → `/booking` | FAQから予約ページへ遷移できる |
| F-011 | 全ヘッダーナビゲーションを巡回する | 任意ページから Catalog / Experience / Booking / Facility / Safety / Sell Memory / FAQ / About / Contact を順にクリック | すべてのリンク先が正常表示される |
| F-012 | 存在しないURLから復帰する | `/unknown-path` を開く → ヘッダーナビをクリック | トップページ相当の画面が表示され、ナビゲーションで目的ページへ移動できる |
| F-013 | 存在しない詳細URLから復帰する | `/template/not-found` または `/episode/not-found` を開く → カタログ内リンクをクリック | カタログページ相当の画面から有効な詳細ページへ移動できる |
| F-014 | チャットから各ページ情報を確認する | 任意ページでチャットを開く → FAQ、予約、安全性などを質問する | 回答が表示され、チャットの開閉・送信状態が破綻しない |
| F-015 | スマホ幅で主要導線を確認する | スマホ幅で `/` → `/catalog` → 詳細 → `/booking` と進む | 横スクロールや重なりがなく、CTAをタップできる |

## 3. 優先度の目安

| 優先度 | 対象 |
|---|---|
| 高 | トップ、カタログ、予約、問い合わせ、主要ナビ、予約完了導線 |
| 中 | 詳細ページ全slug、施設動画、FAQ開閉、キャンペーン導線 |
| 低 | リンクなしCTAのデモ通知、チャットの詳細回答品質、存在しないURLの復帰 |

## 4. 自動テスト化する場合の候補

| 種別 | 候補 |
|---|---|
| 画面表示テスト | 既存のPlaywright visual testを拡張して全ページのスクリーンショットを取得する |
| 導線E2E | 予約完了、問い合わせ完了、カタログ詳細遷移、キャンペーン遷移をPlaywrightで自動化する |
| コンポーネント単体 | Button、Header、FAQ details、FloatingChatWidget、予約フォームの状態変化をテストする |
| 入力検証 | BookingPageとContactPageの必須入力、メール形式、クエリ反映をテストする |
